🏗️ ШПАРГАЛКА: Terraform

📌 Для чего нужен Terraform?
Terraform — это инструмент для создания инфраструктуры через код (IaC — Infrastructure as Code).
Вместо того чтобы нажимать кнопки в облачной консоли, ты описываешь сервера, сети, диски в файлах, и Terraform сам создаёт их в облаке (AWS, Yandex Cloud, GCP и других).

Основные понятия

ТерминЧто это
ProviderПровайдер облака (AWS, Yandex Cloud, GCP)
ResourceРесурс, который создаётся (сервер, диск, сеть)
StateФайл terraform.tfstate — здесь Terraform хранит информацию о созданных ресурсах
PlanПоказывает, что будет создано или изменено (без реальных действий)
ApplyСоздаёт или изменяет ресурсы в облаке
DestroyУдаляет все ресурсы, созданные Terraform
VariableПеременная, чтобы не хардкодить значения (ключи, регион)
OutputВыводит полезную информацию после apply (например, IP адрес)
Security GroupПравила доступа к серверу (файервол в AWS)
Key PairSSH-ключ для подключения к серверу

Основные команды Terraform

terraform init # инициализация (скачать провайдера)
terraform plan # показать план изменений
terraform apply # создать/изменить инфраструктуру
terraform destroy # удалить всё, что создал Terraform
terraform state list # список ресурсов в состоянии
terraform state show <resource_name> # подробная информация о ресурсе
terraform state rm <resource_name> # удалить ресурс из состояния
terraform output # показать выходные переменные
terraform destroy -target <resource_name> # удалить только конкретный ресурс

Пример конфигурации (main.tf) для AWS

terraform { required_providers { aws = { source = "hashicorp/aws" version = "~> 5.0" } } } provider "aws" { region = "us-east-1" } resource "aws_key_pair" "devops_key" { key_name = "devops-key" public_key = file("~/.ssh/id_ed25519.pub") } resource "aws_security_group" "allow_ssh" { name = "allow_ssh" description = "Allow SSH inbound traffic" ingress { from_port = 22 to_port = 22 protocol = "tcp" cidr_blocks = ["0.0.0.0/0"] } egress { from_port = 0 to_port = 0 protocol = "-1" cidr_blocks = ["0.0.0.0/0"] } } resource "aws_instance" "devops_server" { ami = "ami-0c02fb55956c7d316" instance_type = "t3.micro" key_name = aws_key_pair.devops_key.key_name vpc_security_group_ids = [aws_security_group.allow_ssh.id] tags = { Name = "Terraform-VM" } } output "public_ip" { value = aws_instance.devops_server.public_ip }

❌ Реальные ошибки и их решения

1. Ошибка: Error: Invalid provider registry host
Причина: Нет доступа к официальному реестру Terraform (блокировка, DNS).
Решение: Использовать зеркало, добавив в ~/.terraformrc:
provider_installation { network_mirror { url = "https://terraform-mirror.yandexcloud.net/" } }
2. Ошибка: Error: No valid credential sources found
Причина: Terraform не видит ключи AWS.
Решение: Передать ключи через переменные окружения:
export AWS_ACCESS_KEY_ID="AKIA..."
export AWS_SECRET_ACCESS_KEY="..."
3. Ошибка: Error: InvalidParameterCombination: The specified instance type is not eligible for Free Tier
Причина: Выбранный тип инстанса не входит в бесплатный тариф.
Решение: Заменить instance_type на t3.micro или t2.nano.
4. Ошибка: Permission denied (publickey)
Причина: Неправильный SSH-ключ или пользователь.
Решение: Проверь публичный ключ в AWS Console и локальный ключ. Попробуй пользователя ec2-user (для Amazon Linux) или ubuntu (для Ubuntu).
5. Ошибка: Error: creating EC2 Instance: operation error EC2: RunInstances ... VPCIdNotSpecified
Причина: Не указана Security Group или VPC.
Решение: Добавь vpc_security_group_ids = [aws_security_group.allow_ssh.id] в конфиг инстанса.
6. Ошибка: Error: No configuration files
Причина: Ты выполнил terraform plan не в папке с main.tf.
Решение: Перейди в папку с конфигом (cd ~/aws-terraform).

Полезные флаги

  • -target=resource_name — применить изменения только к одному ресурсу
  • -auto-approve — не спрашивать подтверждения (осторожно!)
  • -var="key=value" — передать переменную

Главные правила Terraform

💡 1. Всегда запускай terraform plan перед terraform apply.
Это позволит увидеть, что изменится, и избежать неожиданных сюрпризов.
💡 2. Храни terraform.tfstate в безопасности.
Это файл состояния — в нём хранится информация о созданных ресурсах. Если потеряешь его, Terraform не сможет управлять инфраструктурой.
💡 3. Не храни секреты (ключи) в main.tf.
Используй переменные окружения или terraform.tfvars.
