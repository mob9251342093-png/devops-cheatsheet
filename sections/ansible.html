⚙️ ШПАРГАЛКА: Ansible

📌 Для чего нужен Ansible?
Ansible — это инструмент для автоматизации настройки серверов.
Вместо того чтобы вручную заходить на каждый сервер по SSH и выполнять команды, ты пишешь плейбук (инструкцию), и Ansible сам всё делает.

Основные понятия

ТерминЧто это
InventoryФайл со списком серверов (где выполнять команды)
PlaybookИнструкция для Ansible (что делать на серверах)
TaskОдно действие (установить пакет, скопировать файл)
ModuleГотовая функция (apt, service, copy, cron)
HandlerЗадача, которая выполняется только при изменении (например, перезапустить nginx)

Команды Ansible

ansible --version # версия
ansible all -i inventory.ini -m ping # проверить доступность серверов
ansible all -i inventory.ini -m shell -a "uptime" # выполнить команду на всех серверах
ansible-playbook -i inventory.ini playbook.yml # запустить плейбук
ansible-playbook -i inventory.ini playbook.yml --check # проверить (dry-run) без изменений
ansible-playbook -i inventory.ini playbook.yml -v # подробный вывод (verbose)

Пример инвентарного файла (inventory.ini)

[webservers] localhost ansible_connection=local [all:vars] ansible_python_interpreter=/usr/bin/python3

Пример плейбука (install-nginx.yml)

--- - name: Установка и запуск nginx hosts: webservers become: yes tasks: - name: Установить nginx apt: name: nginx state: present update_cache: yes - name: Запустить nginx service: name: nginx state: started enabled: yes

Пример плейбука для бэкапа (deploy-backup.yml)

--- - name: Настройка резервного копирования hosts: webservers become: yes tasks: - name: Создать папку для скриптов file: path: /opt/backup-scripts state: directory mode: 0755 - name: Скопировать скрипт copy: src: /home/devops/devops_notes/backup.sh dest: /opt/backup-scripts/backup.sh mode: 0755 - name: Добавить задание в cron cron: name: "Ежедневный бэкап" minute: "0" hour: "3" job: "/opt/backup-scripts/backup.sh >> /var/log/backup.log 2>&1"

❌ Реальные ошибки и их решения

1. Ошибка: Unable to parse inventory.ini as an inventory source
Причина: Неправильный синтаксис в инвентарном файле или неверное имя файла.
Решение: Убедись, что файл существует и имеет правильный синтаксис. Используй cat inventory.ini для проверки.
2. Ошибка: sudo: требуется указать пароль
Причина: Ansible не может выполнять команды с sudo без пароля.
Решение: Добавь в инвентарный файл пароль или разреши пользователю выполнять sudo без пароля через sudo visudo.
3. Ошибка: cp: не удалось выполнить stat для '/app/*.sh': Нет такого файла или каталога
Причина: Путь внутри скрипта не соответствует реальной структуре сервера.
Решение: Проверь, откуда скрипт копирует файлы. Если скрипт писался для Docker, путь должен быть изменён на реальный.
4. Ошибка: crontab -l | grep backup не показывает задание
Причина: Ты проверяешь cron пользователя devops, а задание добавлено в cron root.
Решение: Используй sudo crontab -l | grep backup.
5. Ошибка: Permission denied при записи в папку
Причина: Недостаточно прав для создания файлов или записи в системные папки.
Решение: Используй sudo для команд, требующих прав, или измени владельца папки (chown).

Главное правило Ansible

💡 Если ты делаешь что-то на сервере вручную чаще одного раза — напиши плейбук.
Это сэкономит часы работы и избавит от ошибок.
