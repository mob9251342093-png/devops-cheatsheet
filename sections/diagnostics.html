🐞 Диагностика системы: strace и lsof

📌 Зачем это нужно?
strace — показывает, какие системные вызовы делает программа.
Когда приложение падает или не запускается, strace помогает понять, что пошло не так (например, не хватает прав, нет файла, занят порт).

lsof — показывает, какие файлы и порты открыты процессами.
Помогает быстро найти, кто занял порт, или какие файлы использует процесс.

Команды strace

strace -p <PID> # Подключиться к процессу
strace -e trace=network curl # Только сетевые вызовы
strace -e trace=file ls # Только операции с файлами
strace -e trace=open,read,write command # Конкретные вызовы
strace -o output.log command # Сохранить вывод в файл

Команды lsof

lsof -p <PID> # Файлы процесса
sudo lsof -i -P -n # Все открытые порты
sudo lsof -i :80 # Кто слушает порт 80
sudo lsof -u devops # Файлы пользователя devops

Разбор вывода lsof

ПолеЧто значит
COMMANDИмя процесса
PIDID процесса
USERПользователь, запустивший процесс
FDТип дескриптора (cwd, txt, mem, 0u, 1u, 2u)
NAMEФайл или ресурс

Типы дескрипторов (FD)

  • cwd — текущая рабочая директория
  • rtd — корневая директория (/)
  • txt — исполняемый файл
  • mem — библиотека в памяти
  • 0u, 1u, 2u — stdin, stdout, stderr

Главное правило

💡 Если приложение не запускается — сначала проверь порты через lsof -i.
Если порт свободен — используй strace -o log.txt command и читай вывод.
