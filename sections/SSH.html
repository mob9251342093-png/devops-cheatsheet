🔒 ШПАРГАЛКА №6: SSH

Подключение к серверам и управление ключами

Основные команды

ssh-keygen -t ed25519 -C "email@example.com"
cat ~/.ssh/id_ed25519.pub
ssh user@192.168.1.1
scp file.txt user@192.168.1.1:/remote/
ssh-copy-id user@192.168.1.1
eval "$(ssh-agent -s)" && ssh-add ~/.ssh/id_ed25519

Когда что использовать

ЗадачаКомандаПочему
Создать ключssh-keygen -t ed25519ed25519 — безопаснее, чем RSA
Подключиться к серверуssh user@ipСтандартный способ
Копировать файл на серверscp file user@ip:/path/Безопасное копирование
Добавить ключ на серверssh-copy-id user@ipУпрощает подключение без пароля
💡 Главное правило: Публичный ключ можно показывать всем. Приватный ключ (id_ed25519) — храни в секрете и никогда не копируй на сервер.
